Lua Integration in Octopus

Lua is a powerful, efficient, lightweight, embeddable scripting language that serves as the core scripting engine in Octopus. It provides dynamic configuration, custom logic execution, and extensible automation capabilities.

Key Features in Octopus:

  • Dynamic script execution at runtime
  • Custom business logic implementation
  • Configuration-driven automation
  • Seamless Go-Lua interoperability

    eocto.setLocal("currentPage",paths[path])

Debug Functions

eocto.debug(level, message)

Logs debug messages with different severity levels

eocto.debug("info", "User logged in successfully")
eocto.debug("warning", "Rate limit approaching")
eocto.debug("error", "Database connection failed")

Session Management

eocto.getSession(key)

Retrieves a value from the user session

local userId = eocto.getSession("user_id")

eocto.setSession(key, value)

Stores a value in the user session

eocto.setSession("user_id", "12345")

eocto.deleteSession(key)

Removes a specific key from the session

eocto.deleteSession("temp_data")

eocto.setSessionExpiry(seconds)

Sets session expiration time in seconds

eocto.setSessionExpiry(3600) -- 1 hour

Cookie Management

eocto.getCookie(name)

Retrieves a cookie value by name

local theme = eocto.getCookie("user_theme")

eocto.setCookie(name, value, options)

Sets a cookie with optional configuration

eocto.setCookie("user_theme", "dark", {maxAge = 86400})

eocto.getAllCookies()

Returns all cookies as a table

local cookies = eocto.getAllCookies()

eocto.deleteCookie(name)

Removes a specific cookie

eocto.deleteCookie("temp_cookie")

eocto.clearAllCookies()

Removes all cookies from the client

eocto.clearAllCookies()

Security & Encryption

eocto.getCsrfToken()

Retrieves the CSRF protection token

local token = eocto.getCsrfToken()

eocto.encryptData(data)

Encrypts sensitive data using server encryption

local encrypted = eocto.encryptData("sensitive_info")

eocto.decryptData(encryptedData)

Decrypts previously encrypted data

local decrypted = eocto.decryptData(encrypted)

HTTP & Request Handling

eocto.getMethod()

Returns the HTTP method (GET, POST, etc.)

local method = eocto.getMethod()

eocto.getPath()

Returns the current request path

local path = eocto.getPath()

eocto.getHost()

Returns the request host/domain

local host = eocto.getHost()

eocto.getSchema()

Returns the request schema (http/https)

local schema = eocto.getSchema()

eocto.getQueryParams()

Returns all query parameters as a table

local params = eocto.getQueryParams()

eocto.getPathParams()

Returns all path parameters as a table

local pathParams = eocto.getPathParams()

eocto.getPathParam(key)

Returns a specific path parameter value

local userId = eocto.getPathParam("id")

eocto.getPostBody()

Returns the POST request body data

local body = eocto.getPostBody()

eocto.makeRequest(options)

Makes HTTP requests to external servers

local response = eocto.makeRequest({url = "https://api.example.com", method = "GET"})

eocto.proxy(options)

Proxies requests to external servers

eocto.proxy({url = "https://api.example.com", method = "POST"})

local headers = eocto.getHeaders()

eocto.getHeader(name)

Returns a specific header value

local userAgent = eocto.getHeader("User-Agent")

eocto.setHeader(name, value)

Sets a response header

eocto.setHeader("Content-Type", "application/json")

eocto.deleteHeader(name)

Removes a specific header

eocto.deleteHeader("X-Custom-Header")

Data Encoding & JSON

eocto.decodeJSON(jsonString)

Parses JSON string into Lua table

local data = eocto.decodeJSON('{"name": "John", "age": 30}')

eocto.encodeJSON(table)

Converts Lua table to JSON string

local json = eocto.encodeJSON({name = "John", age = 30})

eocto.encodeBase32(data)

Encodes data using Base32 encoding

local encoded = eocto.encodeBase32("hello world")

eocto.decodeBase32(encodedData)

Decodes Base32 encoded data

local decoded = eocto.decodeBase32(encoded)

MongoDB Operations

eocto.getDataFromCollection(collection, query)

Retrieves documents from MongoDB collection

local users = eocto.getDataFromCollection("users", {active = true})

eocto.setDataToCollection(collection, filter, update)

Updates documents in MongoDB collection

eocto.setDataToCollection("users", {id = "123"}, {name = "John Doe"})

eocto.delDataFromCollection(collection, filter)

Deletes documents from MongoDB collection

eocto.delDataFromCollection("users", {inactive = true})

eocto.insertDataToCollection(collection, document)

Inserts new documents into MongoDB collection

eocto.insertDataToCollection("users", {name = "Jane", email = "jane@example.com"})

Redis Cache Operations

eocto.getRedis(key)

local cachedData = eocto.getRedis("user:123:profile")

eocto.setRedis(key, value, expiration)

Stores a value in Redis cache with optional expiration

eocto.setRedis("user:123:profile", userData, 3600)

eocto.deleteRedis(key)

Removes a key from Redis cache

eocto.deleteRedis("user:123:session")

WebSocket Support

WebSocket Route Configuration

Octopus supports WebSocket connections through route configuration. WebSocket routes are automatically handled with session management and client tracking.

-- WebSocket routes are configured in module configuration
-- and provide real-time bidirectional communication
-- with automatic client management and broadcasting
Real-time Communication Client Management Broadcasting

Route & Module Configuration

Pre-check Scripts

Routes can have pre-check scripts that execute before the main route handler. These scripts are defined in the route configuration and run as middleware.

-- Pre-check scripts are executed as middleware
-- and can modify request context, validate data,
-- or perform authentication checks

Static File Serving

Octopus automatically configures static file serving for each module, including CSS, JavaScript, images, fonts, and icons.

-- Static paths are automatically configured:
-- /static, /css, /js, /img, /images, /fonts, /icons

Session & CSRF Integration

All routes automatically include session management and CSRF protection middleware for security.

-- Sessions and CSRF tokens are automatically
-- managed for all routes and available in templates
